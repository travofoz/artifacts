01 // The Philosophy: Fearless Ephemerality

The Problem: The current state of the art demands premium hardware for agent-driven development. Heavy MacOS instances, bloated runtimes, and local environments that degrade over time as autonomous agents experiment, download dependencies, and inevitably trash the system.

The Solution: Project lifecycle versus permanence. This stack treats developer environments as disposable. By isolating everything inside rootless Podman containers on a bare-metal Void Musl host, we achieve fearless execution. If an Open Code or AntiGravity agent goes rogue or breaks the dependency tree, it doesn't matter. You either restore from a Checkpoint/Restore In Userspace (CRIU) snapshot, or nuke the container and rebuild it from the Containerfile in seconds.

You don't need a $2,000 laptop. You can run this on a potato. The host does nothing but provide a kernel and a display. The containers do all the heavy lifting.

02 // Architecture & State Mechanics

The Host Layer

Void Linux + Musl + i3: Near-zero overhead. The host runs no daemons, no heavy standard libraries, and no bloat. It exists solely to pass compute power to the container layer.

The Glibc Execution Layer

Rootless Podman: Full Void Glibc containers acting as an "App Store." VS Code, Python/uv, Node/pnpm, PHP, and C/C++ toolchains run in isolated, daemonless environments without fighting host library incompatibilities.

Shared Storage & Hibernation

  • Universal Bindings: A central /workspace directory mounted across all containers. Agents and IDEs read/write the same code with zero sync lag.
  • Content-Addressable Caches: Global volumes for uv and pnpm prevent duplicate dependency downloads across ephemeral containers.
  • CRIU Checkpointing: Freeze active containers, dump RAM to disk, and resume later. Warning: Headless agents hibernate flawlessly, but X11 graphical tools (VS Code) will drop socket connections. Perform a clean workspace shutdown for IDE containers before checkpointing.

03 // Production Code Blocks

config.sh (Declarative Pathing)

#!/bin/sh
# System-wide declarative definitions. Native sourcing, no YAML overhead.
export HOST_WORKSPACE="/home/travis/workspace"
export CONTAINER_RUNTIME="podman"
export GLOBAL_CACHE_DIR="/home/travis/.cache/global"
export TARGET_ARCH=$(uname -m)

# GUI flags for X11 forwarding
export ENABLE_GUI_SOCKETS=true

boot.sh (Zero-Dependency Provisioner)

#!/bin/sh
source ./config.sh

echo "[*] Injecting xbps non-free repositories..."
sudo xbps-install -Su void-repo-nonfree

echo "[*] Configuring systemd-timesyncd / chrony alternative..."
sudo xbps-install -y chrony
sudo ln -s /etc/sv/chronyd /var/service/

echo "[*] Scaffolding workspace and caches..."
mkdir -p "$HOST_WORKSPACE" "$GLOBAL_CACHE_DIR/uv" "$GLOBAL_CACHE_DIR/pnpm"
chmod 777 "$HOST_WORKSPACE" # Fast local dev mapping

echo "[*] Host provisioned. Ready for container orchestration."

Containerfile (Void Glibc Base)

FROM voidlinux/voidlinux-glibc:latest

# Inject core toolchains
RUN xbps-install -Syu && \
    xbps-install -y base-devel python3 nodejs php git curl xauth

# Create non-root developer identity
RUN useradd -m -s /bin/bash devuser
USER devuser
WORKDIR /workspace

# Set environments for global cache mapping
ENV UV_CACHE_DIR=/global-cache/uv
ENV PNPM_HOME=/global-cache/pnpm
ENV PATH="$PNPM_HOME:$PATH"

CMD ["/bin/bash"]

compose.yaml (Orchestration & X11)

version: '3.8'
services:
  agent-harness:
    build: 
      context: .
      dockerfile: Containerfile
    volumes:
      - ${HOST_WORKSPACE}:/workspace:z
      - ${GLOBAL_CACHE_DIR}:/global-cache:z
    security_opt:
      - seccomp=unconfined # Required for certain agent sandboxing

  ide-gui:
    image: my-vscode-glibc:latest
    environment:
      - DISPLAY=${DISPLAY}
    volumes:
      - /tmp/.X11-unix:/tmp/.X11-unix:ro
      - ${HOST_WORKSPACE}:/workspace:z
    network_mode: host

Hibernation Executables (Terminal Snippets)

# Dump agent context to disk and kill container
podman container checkpoint agent-harness --export=/workspace/backups/agent-snap.tar.gz

# Restore agent context into active RAM
podman container restore --import=/workspace/backups/agent-snap.tar.gz

"Investigate and map the memory paging behaviors, file descriptor serialization, and socket tracking mechanisms inside the Linux CRIU framework during an active container dump. "Evaluate the exact compilation flag disparities, memory layout variations, and binary compatibility fault lines when linking shared objects between a Void Musl host and a Void Glibc container.
